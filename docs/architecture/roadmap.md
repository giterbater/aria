# ARIA v1 Roadmap

ARIA v0.x built the core organs: cognition, language, memory, reasoning, execution, reflection, learning, and skills.

ARIA v1 should be judged by measurable improvement, not module count.

## Phase A: Benchmarking

Priority: highest.

Every build should answer:

- Is reasoning better?
- Is planning faster?
- Is memory retrieval improving?
- Is language understanding improving?
- Is execution more reliable?

Initial benchmark categories:

- reasoning quality
- plan validity
- execution reliability
- memory retrieval relevance
- language understanding accuracy
- learning impact on repeated tasks
- ARIA World survival, happiness, knowledge, economy, growth, culture, and innovation

Visualization is part of benchmarking. The ARIA World dashboard makes population, trust, resources, knowledge, culture, innovations, and scores visible while the system evolves.

## Phase B: Internal Cognitive State

Add internal variables that influence planning and decision-making:

- confidence
- uncertainty
- curiosity
- persistence
- caution
- workload
- novelty

These should be functional control signals, not personality theater.

## Phase C: Better Reasoning

Move from direct planning to option evaluation:

```text
Understand
Generate hypotheses
Evaluate options
Choose
Execute
Verify
```

## Phase D: Multi-Agent Testing

Run several ARIA instances against shared problems to test coordination, critique, and diversity of approach.

This phase is about evaluation and robustness, not civilization.

## Phase E: Civilization

Only pursue this after benchmarks show that individual ARIA capabilities improve reliably and measurably.

## Near-Term Rule

No major architectural feature should land without a benchmark or measurable success criterion attached.
